# **Updated User Flow (v2.0)**

## **1\. Admin: Recording a Stringing Service**

1. **Search**: Admin enters Customer Phone Number.  
2. **Profile**: System opens Customer Profile.  
3. **Action**: Admin clicks "Record Stringing Service".  
4. **Input**:  
   * Select Date (Default: Today).  
   * Select Racket Brand (Yonex/Victor/etc.) \-\> Filtered Racket Model List.  
   * Select String Brand \-\> Filtered String Model List.  
   * Select Tension (Quick Chip or Manual).  
5. **Complete**: System saves to Arsenal and Transaction history.

## ---

**2\. Customer: Viewing the "Arsenal"**

1. **Login**: Customer opens their dashboard.  
2. **Navigate**: Clicks "My Arsenal / 武器库".  
3. **View Stats**:  
   * See **Tension Trend Line** (Is my tension increasing?).  
   * See **String Popularity** (Which string do I use most?).  
4. **View Rackets**:  
   * Check specific racket (e.g., Yonex Astrox 99).  
   * "Strung 20 days ago with BG66UM @ 27lbs. Status: Healthy."

## ---

**3\. Core Product Loop (Badminton Edition)**

**Service \-\> Log \-\> Analyze \-\> Notify \-\> Return**

* Admin provides professional service and logs data.  
* App provides technical value to the user (Arsenal/Trends).  
* App notifies user when strings are likely to have lost tension (Fatigue).  
* User returns to maintain their "Weaponry".
